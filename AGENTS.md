# courier AGENTS

## 文件约定

- 运行 `just` 查看所有可用命令（Go 工具链：`just go ...`，示例服务：`just example ...`）
- 项目背景和模块索引见 `README.md` 和 `docs/`
- 行为约束见本文档

## 开发环境

- Go ≥ 1.26（见 `go.mod`）
- 无需额外初始化，直接 `just go test`、`just go vet` 等入口

## 编码风格

- 文档、注释、错误和测试说明使用中文，专有名词除外
- 先做最小盘点再开始改动
- 不做与任务无关的功能扩展、顺手重构或批量风格清理
- 不因目录已存在而默认新增模块、执行入口或注册关系
- 优先通过参数传递上下文，避免可变全局状态
- 保持基本架构分层，不为局部便利破坏长期一致性
- 能完成最小验证时先验证再交付；无法验证时说明缺口

## 什么时候停、什么时候问

- 需求边界不清且存在多种互斥实现时停下
- 变更会影响现有生成行为但无法完成最小验证时先确认
- 需要仓库外权限、敏感配置或发布动作时请求人工处理
- 需要决定是否新增模块、生成器或执行入口时问人

## 规则生效

- 只有落盘到 `AGENTS.md` 的规则才生效
- 子目录 `AGENTS.md` 只覆盖确有必要的局部规则，未覆盖的继承上级
